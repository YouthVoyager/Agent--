# ADR 0003: 网关内部采用至少一次投递

## 状态

Accepted

## 背景

遥测网关会遇到后端不可用、网络超时、进程重启、WAL 恢复和配置切换。若要求每条数据端到端只写入一次，系统复杂度会明显上升，并且需要后端协同。

## 决策

网关内部采用至少一次投递：

- 接收成功后按当前模式进入内存队列或 WAL；
- 导出失败可重试；
- WAL 恢复后继续发送未确认批次；
- 下游可能收到重复批次。

## 后果

优点：

- 可靠性边界清晰；
- 实现复杂度和 V1 目标匹配；
- 与常见遥测后端重试模型兼容。

代价：

- 下游需要接受重复写入，或在 exporter/后端层做幂等与去重；
- 指标、日志和 trace 的重复影响需要分别评估。
