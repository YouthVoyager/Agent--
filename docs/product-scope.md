# 产品边界

## 做什么

Policy-first Telemetry Gateway 是一个多租户遥测数据网关，面向多个业务团队统一接入 traces、metrics、logs，并在发送到后端前执行认证、租户识别、策略处理、路由、限流、缓冲和回放。

V1 的重点是：

- 租户身份来自认证结果；
- 策略匹配和动作执行可审计；
- 配置支持检查、热更新和回滚；
- 路由支持单路发送和多后端 fanout；
- 后端不可用时提供有界内存队列，关键路由可进入 WAL；
- 暴露自身健康检查、就绪检查、metrics 和 pprof。

## 不做什么

V1 不做完整 Prometheus 服务发现、完整 scrape scheduler、tail-based sampling、完整 Envoy xDS 控制面、Web UI、Wasm 插件、自研通用查询存储、几十种 exporter、跨地域共识或分布式 WAL。

项目也不定位为通用可观测性平台，不和 OpenTelemetry Collector 竞争组件数量。

## 租户身份

租户身份必须来自 API Key、mTLS 或其他认证结果。客户端 payload 或 header 中声明的租户字段只能作为普通属性处理，不能覆盖认证解析出来的 `tenant_id`。

## ACK 含义

非持久模式下，ACK 表示数据已经进入当前进程的内存队列；进程崩溃可能造成队列内数据丢失。

持久模式下，ACK 表示数据已经写入 WAL，并满足当前配置的同步策略；ACK 不表示后端已经完成写入。

## 重复和丢失

网关内部目标是至少一次投递。进程重启、导出重试、后端超时和 WAL 恢复都可能让后端收到重复批次。

非持久队列在进程崩溃时可能丢失尚未导出的数据。持久模式仍需要关注磁盘故障、WAL 损坏、配置错误和运维误删。
